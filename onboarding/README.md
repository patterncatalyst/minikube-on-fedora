# Onboarding

If you're new to the repo and trying to figure out how to get
oriented, this is the right place to start. The three documents
in this folder cover three different angles on the same
question — "how do I work on this thing?" — for three different
kinds of contributors.

## What to read first, based on what you're doing

**You want to set up the repo locally and run the site or the
examples** →
[`GETTING-STARTED.md`](GETTING-STARTED.md). Tools, versions,
typical first-time gotchas. Read this first if you've just
cloned the repo and you're staring at it.

**You want to understand why certain decisions look the way they
do, or you want to avoid the same mistakes we already made** →
[`LESSONS-LEARNED.md`](LESSONS-LEARNED.md). Empirical guidance,
pitfalls, the "why" behind the conventions in `CONTRIBUTING.md`.

**You want to continue the human-AI collaboration pattern that
produced this tutorial (or replicate the workflow for a
different project)** →
[`STARTING-WITH-CLAUDE.md`](STARTING-WITH-CLAUDE.md). How the
back-and-forth was structured, what worked, what to avoid.

## What else is worth reading

These live at the repo root, not in `onboarding/`, but they're
adjacent reading:

- [`../README.md`](../README.md) — top-level repo overview;
  what's where, how to run, where to find things
- [`../CONTRIBUTING.md`](../CONTRIBUTING.md) — coding/writing
  conventions for this repo (UBI images, idempotent demo
  scripts, single-line shell commands, etc.)
- [`../PRD.md`](../PRD.md) — the product requirements doc; the
  intent against which the tutorial was built
- [`../_plans/reconciliation-plan.md`](../_plans/reconciliation-plan.md)
  — audit trail of every iteration; what shipped, what was
  retracted, what was learned
- [`../_plans/prd-reconciliation.md`](../_plans/prd-reconciliation.md)
  — what shipped vs. what was planned, with rationale for
  every divergence
